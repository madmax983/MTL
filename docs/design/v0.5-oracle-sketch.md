# MTL v0.5 — Oracle sketch (INPUT MATERIAL, verbatim)

- This is **input material**: a Gemini sketch the owner ran, committed **verbatim** for the record. It is not a design decision. Its `fork`-as-primitive framing is **REJECTED** — speculation is host-layer only (see `v0.5-refactor.md` §4), because a `fork` word inside the evaluation relation would break the proved P1 determinism theorem. Its continuation representation (`cont: QuoteId`) is **under-specified** — a bare `QuoteId` cannot express a partially-consumed continuation nor O(1) prepend; the design doc's segment-cursor continuation (`v0.5-refactor.md` § Continuation) closes that gap.

---

Multiplicity is exactly where this architecture begs to go next. You already have a fully reified, verified execution state (Vm { stack, cont }). Taking that and scaling it out geometrically is where the concatenative model really leaves traditional AST-based interpreters in the dust. Because your state is strictly bounded and explicitly defined, extending into multiple concurrent states is practically native.

Shape 1 — Speculative Branching (The Multiverse): since VM state can be deeply cloned, implement native non-determinism for search spaces. An agent emits a fork that clones the VM state; explore dozens of paths concurrently, cull branches that Fault, collapse to the sequence that successfully halts. Tree-of-Thought pushed down into the VM layer at hardware speed.

Shape 2 — The Actor Model (The Swarm): pass continuations between concurrent MTL instances mapped onto Rust threads/async tasks with channels. One orchestrator VM, lightweight workers; code is data (quotations), so a VM can pass an executing block of logic to another VM over a channel.

Best of both: the actor model provides infrastructure, speculative branching provides logic. Thousands of lightweight VMs; any individual VM forks itself; manage valid states across diverging paths, drop dead branches, commit successful ones. Main architectural puzzle: memory pressure — deep-cloning vectors for thousands of speculative branches triggers allocator churn.

Memory layer to eliminate deep copies:
1. Structural sharing (persistent stack): operations only happen at the end of the stack; back it with a persistent structure (radix tree or immutable cons-list arena). Fork = O(1) pointer copy; push = new node pointing at old head; memory structurally shared across branches.
2. Global quote interning: quotations are immutable once constructed; a global QuoteArena owns the Vec<Word> arrays; a Quote on the stack becomes a u32 index/fat pointer. Branch = copying a 32-bit integer; Value becomes small and trivially Copy.
3. Generational bump arenas: short-lived allocations from a bump allocator (bumpalo) tied to a search-space generation; allocation = pointer advance; when the search resolves, extract the result and drop the whole arena; no individual frees.

Draft types:

```rust
/// A lightweight, Copy-able handle to a quotation in the arena.
#[derive(Clone, Copy, Debug, PartialEq, Eq)]
pub struct QuoteId { pub start: u32, pub len: u32 }

#[derive(Clone, Copy, Debug, PartialEq, Eq)]
pub enum Word { PushInt(i64), PushQuote(QuoteId), Prim(SpecPrim), Call(u32) /* interned host capability names */ }

#[derive(Clone, Copy, Debug, PartialEq, Eq)]
pub enum Value { Int(i64), Quote(QuoteId) }

pub struct QuoteArena { tape: Vec<Word> }
impl QuoteArena {
    pub fn alloc(&mut self, words: &[Word]) -> QuoteId { let start = self.tape.len() as u32; let len = words.len() as u32; self.tape.extend_from_slice(words); QuoteId { start, len } }
    pub fn cat(&mut self, a: QuoteId, b: QuoteId) -> QuoteId { let start = self.tape.len() as u32; self.tape.extend_from_within(a.start as usize..(a.start+a.len) as usize); self.tape.extend_from_within(b.start as usize..(b.start+b.len) as usize); QuoteId { start, len: a.len + b.len } }
}

pub struct StackPtr(u32);
pub struct StackNode { pub value: Value, pub parent: StackPtr }
pub struct StackArena { nodes: Vec<StackNode> } // index 0 = empty-stack sentinel
// push: new delta node pointing at current anchor; pop: move pointer to parent.

#[derive(Clone, Copy)]
pub struct VmState { pub stack: StackPtr, pub cont: QuoteId }
```

With arenas owning the data, forking = copying two 32-bit integers; spawn 100,000 speculative branches and memory doesn't budge until timelines diverge; a faulted branch is just a dropped VmState; arenas reset when a search generation concludes.
